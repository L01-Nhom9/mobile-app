Getting Started With ClassTrack.

ClassTrack is the essential mobile solution for Instructors and Students to modernize attendance and absence management. It replaces fragmented manual processes (email, messages, paper forms) with a centralized and real-time digital workflow.

Centralized Management & Auditing.

Eliminate scattered requests. Instructors use a single Approval/Denial board where every action is logged, ensuring full traceability and accountability.

Standardized & Error-Free Data Entry.

Students submit requests via a structured Form with mandatory fields and one proof document. This standardization minimizes input errors and ensures data quality.

Full Transparency for Students.

Students can view their complete request history and real-time status (Pending/Approved/Denied). This reduces off-system inquiries and builds user trust.

Effortless Compliance Reporting.

Instructors can quickly export a CSV report detailing student attendance and leave status, enabling rapid compilation of diligence scores and simplified reporting.

How the App Works.

Sign Up & Access

Users (Students and Instructors) log in to access their personalized Home page, which serves as the hub for navigating all ClassTrack features, including class management and attendance workflows.

Manage Classes (Instructor/Student)

Instructors create a class by inputting details; the system then generates a unique class code and password. Students use this code and password to join an existing class. Upon successful validation, the user is added to the class and directed to the class details view.

Create Leave Request (Student)

Students select "Leave Request," choose the class and date/session they will miss, provide a reason, and upload supporting files (Image/PDF). The system validates the input, saves the request as Pending, and confirms the submission before directing the student to the Leave History page.

Review and Deny Requests (Instructor)

Instructors access the "Leave Request List," review specific requests (including reasons and documentation), and choose to Approve or Deny. If denying, they must provide a reason. The system updates the request status and refreshes the list.

The Best Features of The App.

ClassTrack digitizes and centralizes attendance and leave management, solving the problem of scattered, manual processes like email and messaging apps.

Role-Based Security

Ensures proper identity authentication and enforces access control according to the user's role (Student/Instructor).

Class Code Management

Creates a standardized data structure by requiring Instructors to create a class code and Students to use it for enrollment, ensuring accurate request linkage.

Standardized Leave Form

Establishes a standard data entry point for requests, requiring class, date, reason, and a single supporting file (Image/PDF), minimizing errors.

Centralized Approval Board

Provides Instructors with a focused interface to process all pending requests, log actions, and update the status instantly, ensuring traceability.

Student History & Status

Ensures transparency and reduces external communication by providing students with a clear, time-sorted history of their requests, status labels, and reasons for denial.

Attendance Reporting

Generates a quantifiable, quick output for grading. Instructors can export a CSV file containing comprehensive data on student absences, approvals, and denials by class and time frame.
